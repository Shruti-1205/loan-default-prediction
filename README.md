# Loan Default Prediction

End to end machine learning project that predicts the probability a loan
applicant will default. The repository contains the training pipeline, a
FastAPI service that serves predictions with SHAP explanations, and a
Streamlit frontend designed for deployment to Streamlit Community Cloud.

## Highlights

- Reproducible scikit-learn Pipeline (one hot encoding, standard scaling,
  class balanced Random Forest).
- Decision threshold tuned on a held out validation split to maximise F1,
  which is important on this imbalanced dataset (roughly 88 percent non
  default).
- FastAPI service returns the prediction, the probability, the active
  threshold, and the top SHAP feature contributions per request.
- Streamlit app with three tabs: live predictions with SHAP waterfall, a
  model performance report, and a what if analysis.
- Automated evaluation artifacts (confusion matrix, ROC curve, precision
  recall curve, threshold sweep, feature importances) generated by
  `evaluate.py` and rendered in the UI.
- Pytest suite exercised by a GitHub Actions workflow on every push.

## Dataset

`Loan_default.csv` contains demographic, financial, and loan level
attributes for roughly 255,000 applications. The target column `Default`
is binary. Class balance is approximately 88 percent non default to 12
percent default.

## Modeling approach

1. Train a scikit-learn Pipeline with a ColumnTransformer for preprocessing
   and a Random Forest classifier (`class_weight="balanced"`).
2. Hold out 20 percent of the data with stratified sampling for evaluation.
3. Score the held out split and sweep the probability threshold from 0.05
   to 0.95. Pick the threshold that maximises F1.
4. Persist the fitted pipeline plus the selected threshold and the full set
   of metrics to disk. The API and the Streamlit app both load these at
   startup.

The decision threshold is the single most important lever on this dataset.
At the default 0.5 cutoff, the classifier almost never predicts the
positive class. The tuned threshold shifts the operating point to a region
with meaningful recall on the default class while keeping precision usable.

## Local setup

```bash
python -m venv .venv
.venv\Scripts\activate              # Windows
# or: source .venv/bin/activate     # macOS or Linux
pip install -r requirements.txt
```

### Optional: retrain the model

The repository ships with a pre trained model. To regenerate it:

```bash
python train.py
```

### Generate evaluation artifacts

```bash
python evaluate.py
```

This writes `metrics.json` and the PNG charts under `evaluation_artifacts/`.

### Run the FastAPI service

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

Swagger UI is available at http://127.0.0.1:8000/docs.

### Run the Streamlit app

```bash
cd streamlit_app
pip install -r requirements.txt
streamlit run streamlit_app.py
```

The app defaults to loading the joblib pipeline directly, which means it
works standalone. To point it at the FastAPI service instead:

```bash
set USE_API=true                    # Windows
set API_URL=http://127.0.0.1:8000
streamlit run streamlit_app.py
```

### Run tests

```bash
pytest tests/ -v
```

## Docker

Build and run the FastAPI service in a container:

```bash
docker build -t loan-default-api .
docker run -p 8000:8000 loan-default-api
```

## Deployment

The easiest deployment path is Streamlit Community Cloud for the frontend
plus Render or Railway for the FastAPI service. The Streamlit app is
self contained (it loads the joblib pipeline directly), so the simplest
public demo only requires Streamlit Community Cloud.

1. Push the repository to GitHub.
2. Visit https://share.streamlit.io and create a new app pointing at
   `streamlit_app/streamlit_app.py`.
3. Leave `USE_API` unset so the app uses the local pipeline.
4. Optional: deploy `app/main.py` to Render or Railway using the provided
   Dockerfile, then set `USE_API=true` and `API_URL` in the Streamlit Cloud
   app settings.

## API reference

- `GET /` returns service metadata including the active threshold.
- `GET /health` returns `{"status": "ok"}`.
- `GET /metrics` returns the contents of `metrics.json`.
- `POST /predict` accepts the applicant payload and returns the
  prediction, the probability of default, the active threshold, the
  risk label, and the top ten SHAP feature contributions.

Example request body is available in the Swagger UI at `/docs`.
